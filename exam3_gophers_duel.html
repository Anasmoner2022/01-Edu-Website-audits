Objective
🎯 What you will prove in this exam

Master the stack as a tool for remembering the past and matching it against the present — and the two-pointer as a tool for eliminating impossible pairs without brute force. These two ideas together solve a surprisingly large fraction of all interview problems.

💡
The core insight: A stack is controlled memory — it remembers things in reverse order and lets you check "does what I see now match what I saw most recently?" The two-pointer eliminates the need to compare everything against everything.

02The Memory Keeper LC #155Medium▶

Design a stack supporting Push, Pop, Top, and GetMin — all in O(1).

  • Use two parallel slices: one for values, one tracking the running minimum at each depth
  • On every Push: append min(newVal, currentMin) to the mins stack
  • Pop both stacks together on every Pop
Expected Type & Functions

func Constructor() MinStack
func (s *MinStack) Push(val int)
func (s *MinStack) Pop()
func (s *MinStack) Top() int
func (s *MinStack) GetMin() int
Constraints
-231 ≤ val ≤ 231-1  |  Pop/Top/GetMin called only on non-empty stack
All ops: O(1)Space: O(n)
Allowed & Forbidden
Allowed
two []int slicesappend()slice shrink
// vals stack + mins stack — both move in sync
Forbidden
sort.Intslinear min scanmap
// No O(n) min search — must be O(1)
💡 Key Insight
mins[i] stores the minimum at depth i and below. Push: append(mins, min(val, mins[top])). Pop: shrink both. GetMin() = mins[top] — always O(1).

  • Keep popping while the condition holds, then push i

}

Allowed & Forbidden
Allowed
[]int index stackappend()result []int
// Stack stores indices — not temperatures
Forbidden

10The Queue Architect LC #232Medium▶

Implement a FIFO queue using only two stacks. Push must be O(1). Pop and Peek must be amortised O(1).

  • inbox: receives all pushes
  • outbox: serves all pops/peeks — refill from inbox only when empty
  • Each element moves inbox→outbox at most once — amortised O(1)
Expected Type & Functions
type MyQueue struct { /* inbox, outbox []int */ }
func NewMyQueue() MyQueue
func (q *MyQueue) Push(x int)
func (q *MyQueue) Pop() int
func (q *MyQueue) Peek() int
func (q *MyQueue) Empty() bool
Usage
func main() {
    q := NewMyQueue()
    q.Push(1); q.Push(2)
    fmt.Println(q.Peek())   // 1
    fmt.Println(q.Pop())    // 1
    fmt.Println(q.Empty()) // false
}
Constraints
Pop/Peek called only on non-empty queue  |  At most 100 calls total
Push: O(1)Pop/Peek: O(1) amortised
Allowed & Forbidden
Allowed
two []int stacksappend()slice shrinkshift() helper
// Pour inbox→outbox only when outbox is empty
Forbidden
container/listchannelsync packagesingle stack
// Two stacks only — no built-in queue structures
💡 Key Insight
Only pour when outbox is empty. If outbox still has elements, pop directly. Each element crosses the boundary at most once — that is the amortised O(1) proof in one sentence.
