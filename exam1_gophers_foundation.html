01 The String Flipper LC #344 Easy ▶

A string arrives as a slice of bytes. Your task: reverse it in-place — no new slice, no helper from the standard library. Swap from both ends, moving inward until the two sides meet.

  • Modify the input slice directly — the function returns nothing.
  • Think of two pointers: one starts at index 0, one at index len-1. They swap and march toward each other.
  • Stop when left and right meet or cross.
Expected Function
func ReverseString(s []byte) {

}
Usage
package main

import "fmt"

func main() {
    s1 := []byte{'h', 'e', 'l', 'l', 'o'}
    ReverseString(s1)
    fmt.Println(string(s1))

    s2 := []byte{'H', 'a', 'n', 'n', 'a', 'h'}
    ReverseString(s2)
    fmt.Println(string(s2))
}
$ go run .
olleh
hannaH
Constraints
1 ≤ s.length ≤ 105
s[i] is a printable ASCII character
Time: O(n) Space: O(1)
Allowed & Forbidden
Allowed
len() index access s[i] temp variable swap for loop
// Two-pointer swap: left and right marching inward
Forbidden
new slice strings.Builder append() copy()
// No extra memory — pure in-place O(1) space
💡 Key Insight
You only need to swap the first half. When left ≥ right, you are done. Draw a 5-character word on paper and trace the swaps before writing any code.
02 The Letter Spy LC #242 Easy ▶

Two words walk in. Are they anagrams? An anagram uses all the same letters, just rearranged. "listen" and "silent" are anagrams. "rat" and "car" are not.

  • Both strings consist of lowercase English letters only.
  • Use a [26]int frequency array — not a map.
  • Count up for one string, count down for the other. Any non-zero slot at the end means not an anagram.
Expected Function
func IsAnagram(s string, t string) bool {

}
Usage
func main() {
    fmt.Println(IsAnagram("anagram", "nagaram"))
    fmt.Println(IsAnagram("rat",     "car"))
    fmt.Println(IsAnagram("listen",  "silent"))
}
$ go run .
true
false
true
Constraints
1 ≤ s.length, t.length ≤ 5×104
s and t consist of lowercase English letters only
Time: O(n) Space: O(1)
Allowed & Forbidden
Allowed
[26]int array byte arithmetic len() range loop
// freq[c - 'a']++ is the key operation
Forbidden
map[byte]int sort.Slice strings package unicode package
// Fixed-size array gives O(1) space — no map needed
💡 Key Insight
In Go, 'a' is a byte with value 97. So c - 'a' maps each letter to an index 0–25 — a perfect fit for a [26]int array without any conversion.

04 The Lost Number LC #268 Easy ▶

A slice contains n distinct numbers in the range [0, n]. Exactly one number is missing. Find it using the sum formula — not a map, not sorting.

  • The expected sum of 0 to n is n × (n+1) / 2.
  • Subtract the actual sum of the slice from this expected sum.
  • The result is the missing number. Understand why this works, not just how.
Expected Function
func MissingNumber(nums []int) int {

}
Usage
func main() {
    fmt.Println(MissingNumber([]int{3, 0, 1}))
    fmt.Println(MissingNumber([]int{0, 1}))
    fmt.Println(MissingNumber([]int{9,6,4,2,3,5,7,0,1}))
}
$ go run .

2
8
Constraints
n = nums.length  |  1 ≤ n ≤ 104
0 ≤ nums[i] ≤ n  |  All numbers in nums are unique
Time: O(n) Space: O(1)
Allowed & Forbidden
Allowed
len() for loop / range arithmetic operators two int variables
// n*(n+1)/2 minus the actual sum
Forbidden
sort.Ints map[int]bool extra slice XOR trick
// Use the sum formula — the intended approach
💡 Key Insight
Gauss, aged 7, summed 1 to 100 instantly: 100 × 101 / 2 = 5050. You are doing the same. The sum formula is one of the most powerful tools in mathematics — not just algorithms.

A slice arrives with unwanted guests — elements equal to val. Remove all of them in-place and return k, the count of elements that remain.

  • Modify nums so the first k elements do not contain val.
  • Return k. Elements beyond index k-1 are irrelevant.
  • Use a write pointer: it only advances when a keeper is found.

06 The Merge Master LC #88 Easy ▶

Two sorted arrays — nums1 and nums2 — must merge into one sorted result stored inside nums1. The trick: nums1 has capacity m + n, with the last n slots zeroed out and available.

  • Merge in-place into nums1. No extra slice.
  • Start from the end. Largest elements go to the back — this avoids overwriting values you still need.
  • m = valid elements in nums1. n = elements in nums2.
Expected Function
func Merge(nums1 []int, m int, nums2 []int, n int) {

}
Usage
func main() {
    nums1 := []int{1, 2, 3, 0, 0, 0}
    Merge(nums1, 3, []int{2, 5, 6}, 3)
    fmt.Println(nums1)

    nums2 := []int{0}
    Merge(nums2, 0, []int{1}, 1)
    fmt.Println(nums2)
}
$ go run .
[1 2 2 3 5 6]
[1]
Constraints
nums1.length == m + n  |  nums2.length == n
0 ≤ m, n ≤ 200  |  Both arrays sorted in non-decreasing order
Time: O(m+n) Space: O(1)
Allowed & Forbidden
Allowed
3 index pointers for loop index assignment comparison operators
// p1=m-1, p2=n-1, tail=m+n-1 — fill from the back
Forbidden
make() new slice append() sort.Ints after copy copy()
// No extra allocation — merge from the back
💡 Key Insight
Filling from the front would overwrite valid values in nums1. Filling from the back is safe — the tail is empty, so nothing useful gets overwritten. The direction of traversal is the entire insight.
07 The Word Sentinel LC #58 Easy ▶

A sentence drifts in — words separated by spaces, possibly trailing silence. The last word holds a secret: its length. Find it.

  • A word is any maximal sequence of non-space characters.
  • Trailing spaces must be ignored — walk backwards past them first.

s consists of English letters and spaces only
At least one word will be present

08 The Carry Keeper LC #66 Easy ▶

A number lives inside a slice — each element is one decimal digit. Add 1 to the number and return the updated array. The challenge is carry propagation: 999 + 1 = 1000.

  • digits[0] is the most significant digit.
  • Walk from the last digit toward the front, adding 1 and propagating carry.
  • If carry exits the front, prepend a 1 — e.g. [9,9,9] becomes [1,0,0,0].

09 The Run-Length Encoder CTCI 1.6 Easy ▶

Compress a string by collapsing consecutive repeated characters into a count followed by the character. This technique — called Run-Length Encoding — is used in image formats (BMP, PCX), fax transmission, and network protocols.

  • "aaabbc" → "3a2b1c" — three a's, two b's, one c.
  • If the encoded result is strictly longer than the original, return the original unchanged.
  • Counts can exceed 9 — handle two-digit counts without strconv.
  • The input contains only lowercase letters.
Expected Function
func RunLengthEncode(s string) string {

}
Usage
func main() {
    fmt.Println(RunLengthEncode("aaabbc"))
    fmt.Println(RunLengthEncode("aaabbbccc"))
    fmt.Println(RunLengthEncode("abcd"))      // strictly longer — return original
    fmt.Println(RunLengthEncode("aaaaaaaaaa")) // 10 a's
}
$ go run .

3a3b3c
abcd
10a
Constraints
1 ≤ s.length ≤ 104
s consists of lowercase English letters only
Return original if encoded form is strictly longer
Time: O(n) Space: O(n)
Allowed & Forbidden
Allowed
strings.Builder byte arithmetic for loop len()
// byte('0' + count%10) writes a digit without strconv
Forbidden
strconv.Itoa fmt.Sprintf strings.Replace regexp package
// Build the output byte by byte — no number conversion
💡 Key Insight
Walk through the string tracking the current character and its run count. When the character changes, write count + char to your builder, then reset count to 1. Don't forget to write the final group after the loop ends.
10 The Caesar Cipher Skiena ADM Easy ▶

Julius Caesar encrypted his military messages by shifting every letter forward by a fixed number of positions in the alphabet. Implement this cipher. It is one of the oldest algorithms in history — and a perfect exercise in modular arithmetic.

  • Shift every letter forward by k positions. Wrap around from z back to a.
  • Non-letter characters (spaces, digits, punctuation) are left unchanged.
  • Preserve case — uppercase letters shift within uppercase, lowercase within lowercase.
  • A shift of 26 (or any multiple) must return the original string unchanged.
Expected Function
func CaesarCipher(s string, k int) string {

}
Usage
func main() {
    fmt.Println(CaesarCipher("abc",         1))
    fmt.Println(CaesarCipher("xyz",         3))  // wraparound
    fmt.Println(CaesarCipher("Hello, World!", 13)) // ROT13
    fmt.Println(CaesarCipher("gopher",      26)) // full rotation = no change
}
$ go run .
bcd
abc
Uryyb, Jbeyq!
gopher
Constraints
0 ≤ s.length ≤ 104
0 ≤ k ≤ 106
s may contain letters, digits, spaces, and punctuation
Time: O(n) Space: O(n)
Allowed & Forbidden
Allowed
byte arithmetic % (modulo) []byte result for loop switch / if
// k = k % 26 first — normalises any large shift
Forbidden
strings.Map unicode package strings.Replace regexp package
// Process each byte manually with a switch statement
💡 Key Insight
Normalise first: k = k % 26. Then for each letter, the shifted index is (ch - 'a' + k) % 26. Adding 'a' back gives you the shifted character. Handle uppercase separately with 'A' as the base. Everything else copies through unchanged.
